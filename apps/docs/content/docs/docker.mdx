Run PostgreSQL with Docker Compose. The Nest API (`apps/nest-api`) runs on the host for now.

## Layout

Compose is split into modular files under **`docker/compose/`** and merged by the root **`docker-compose.yml`** via [`include`](https://docs.docker.com/compose/reference/include/) (requires **Docker Compose v2.20+**).

| Fragment | Contents |
|----------|----------|
| `docker/compose/postgres.yml` | Postgres service, named volume, healthcheck |
| `docker/compose/rust-api.yml` | Optional Rust demo API |

See **`docker/README.md`** at the repo root for the manual `-f` fallback if your Compose build does not support `include`.

## Setup

1. **Copy env file** (from repo root):

   ```bash
   cp env.docker.example .env
   ```

   Edit `.env` if needed (e.g. `POSTGRES_PASSWORD`).

2. **Start Postgres**:

   ```bash
   docker compose up -d
   ```

3. **Start the API on the host**:

   ```bash
   bun --cwd apps/nest-api run dev
   ```

## Ports

| Service | Host port |
|---------|-----------|
| Postgres | 5432 |
